    ```sql
    CREATE TABLE IF NOT EXISTS sessions (
        id              TEXT PRIMARY KEY,
        repo_id         TEXT NOT NULL REFERENCES repos(id) ON DELETE CASCADE,
        worktree_id     TEXT REFERENCES worktrees(id) ON DELETE SET NULL,
        branch          TEXT NOT NULL,
        task_id         TEXT,
        status          TEXT NOT NULL DEFAULT 'active' CHECK(status IN ('active', 'stopped')),
        started_at      INTEGER NOT NULL,
        stopped_at      INTEGER,
        last_activity   INTEGER NOT NULL,
        last_command    TEXT NOT NULL DEFAULT '',
        created_at      INTEGER NOT NULL,
        updated_at      INTEGER NOT NULL
    );
    CREATE INDEX IF NOT EXISTS idx_sessions_repo_id ON sessions(repo_id);
    CREATE INDEX IF NOT EXISTS idx_sessions_worktree_id ON sessions(worktree_id);
    CREATE INDEX IF NOT EXISTS idx_sessions_status ON sessions(status);
    ```

    Create cmd/devctl/session.go following the exact Cobra subcommand pattern from worktree.go:

    **sessionCmd** (parent): `devctl session` — "Manage work sessions"

    **sessionStartCmd** (`devctl session start`):
    - Flags: `--repo/-r` (repo path/name), `--branch/-b` (branch), `--task/-t` (task ID)
    - CWD auto-detection when flags omitted: use `os.Getwd()` to match against worktrees table (exact path match on `w.path`). If no worktree match, try repos table path match. If no DB match, use `git.CurrentBranch(ctx, cwd)` for branch and auto-register the repo via `ensureRepo()` (following pitfall 2 from research).
    - Auto-stop: Before inserting, run `UPDATE sessions SET status='stopped', stopped_at=?, updated_at=? WHERE repo_id=? AND branch=? AND status='active'` to prevent orphaned active sessions (pitfall 1 from research).
    - Insert new session with UUID id, status='active', started_at=now, last_activity=now.
    - Print: `Session started: {id[:8]} ({repoName}/{branch})`
    - Reuse `ensureRepo`, `resolveRepo`, `dbKey{}` from worktree.go (same package main, no import needed).

    **sessionStopCmd** (`devctl session stop`):
    - No flags needed. Auto-detect from CWD: find the active session for the current worktree/repo+branch.
    - `UPDATE sessions SET status='stopped', stopped_at=?, updated_at=? WHERE status='active' AND ...` matching CWD context.
    - Print: `Session stopped: {id[:8]}`
    - If no active session found, print: `No active session for this directory`

    **sessionListCmd** (`devctl session list`):
    - Query all sessions joined with repos, ordered by last_activity DESC.
    - Format output as a table: STATUS REPO/BRANCH LAST_ACTIVITY LAST_COMMAND
    - Active sessions get a green bullet, stopped sessions get a dim bullet.
    - Show relative time for last_activity (e.g., "2m ago", "3h ago", "2d ago").
    - Mark sessions as "stale" if active but last_activity exceeds `session.active_threshold_minutes` viper config (reuse sessionThreshold pattern — but note that function is in internal/dashboard/manager.go, so duplicate the viper read locally or extract to a shared config helper; simplest: just read viper inline).

    **Activity tracking in main.go:**
    - Add `PersistentPostRunE` to the root command (chain with existing `PersistentPreRunE`). The PostRunE fires after every command completes.
    - In PostRunE: get cwd, build full command string from `cmd.CommandPath() + " " + strings.Join(args, " ")`, then `UPDATE sessions SET last_activity=?, last_command=?, updated_at=? WHERE status='active' AND worktree_id IN (SELECT id FROM worktrees WHERE path=?)` using cwd.
    - Also try matching by repo path if worktree match fails (for sessions started at repo root).
    - Errors in PostRunE are non-fatal (log warning, don't fail the command).

    **Wire into main.go:**
    - Add `root.AddCommand(sessionCmd)` alongside existing worktreeCmd, configCmd, repoCmd.
    - Register subcommands in session.go init(): `sessionCmd.AddCommand(sessionStartCmd, sessionStopCmd, sessionListCmd)`
