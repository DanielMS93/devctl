    **Extract AppleScript helpers** (pitfall 3 from research):

    Create `internal/terminal/terminal.go` with the following functions extracted from `pkg/tui/panels/viewer.go`:
    - `RunInNewTerminal(shellCmd string) error` — detects TERM_PROGRAM, dispatches to iTerm2 split or Terminal.app tab (replicating the existing pattern).

    - `iterm2SplitScript(shellCmd string) string` — moved from viewer.go
    - `terminalAppTabScript(shellCmd string) string` — moved from viewer.go

    Update `pkg/tui/panels/viewer.go`:
    - Replace the local `runAppleScript`, `iterm2SplitScript`, `terminalAppTabScript` functions with calls to `internal/terminal` package.
    - `openClaudeInNewWindow` and `openNewClaudeInWindow` should call `terminal.RunInNewTerminal(shellCmd)` instead of the local switch+runAppleScript pattern.
    - Remove the now-dead local functions. Keep `findClaudeBin()` in viewer.go since it's Claude-specific.

    **Create `devctl jump` command** (cmd/devctl/jump.go):

    This is a standalone mini Bubbletea program (NOT part of the dashboard). It runs, user selects, it exits, then opens terminal.

    Data loading (`loadJumpItems`):
    - Query active devctl sessions joined with repos and worktrees: `SELECT s.id, r.name, s.branch, w.path, s.last_activity, s.last_command FROM sessions s JOIN repos r ON r.id=s.repo_id LEFT JOIN worktrees w ON w.id=s.worktree_id WHERE s.status='active' ORDER BY s.last_activity DESC`
    - ALSO include worktrees that have no active devctl session but exist in the DB (so jump is useful even without explicit session start): `SELECT '' as id, r.name, w.branch, w.path, 0 as last_activity, '' as last_command FROM worktrees w JOIN repos r ON r.id=w.repo_id WHERE w.id NOT IN (SELECT worktree_id FROM sessions WHERE status='active' AND worktree_id IS NOT NULL)`
    - Deduplicate by worktree path.

    jumpItem struct implementing list.Item interface:
    - `Title() string` — `repoName / branch` (or `repoName / branch [active]` for sessions)
    - `Description() string` — last_command if available, else worktree path
    - `FilterValue() string` — `repoName branch worktreePath` (all searchable)

    jumpModel struct:
    - `list list.Model` — the bubbles/v2 list component
    - `choice *jumpItem` — set on Enter, triggers tea.Quit
    - `quitting bool`

    Bubbletea v2 API (pitfall 4 from research):
    - `Init() tea.Cmd` (NOT `(tea.Model, tea.Cmd)`)
    - `Update(msg tea.Msg) (tea.Model, tea.Cmd)` — handle `tea.KeyPressMsg` (NOT `tea.KeyMsg`)
    - `View() tea.View` (NOT `string`) — use `tea.NewView()` for alt screen support
    - Enter selects, Esc/q quits without selection
    - Delegate all other keys to list.Update

    List construction: Use `list.New(items, width, height)` with appropriate options. Set title to "Jump to worktree". The exact v2 API for list.New may differ — check the import at `charm.land/bubbles/v2/list`. If the constructor takes different args in v2, adapt accordingly (this is the MEDIUM confidence area from research). The list component provides built-in fuzzy filtering via DefaultFilter.

    After selection (in the Cobra RunE, after tea.Program.Run returns):
    - If `m.choice != nil` and `m.choice.worktreePath != ""`:
      - Build shell command: `cd '{safePath}' && exec $SHELL`
      - Call `terminal.RunInNewTerminal(shellCmd)`
    - If no choice or empty path, exit silently.
    - Fallback for non-macOS or missing osascript: print `cd /path/to/worktree` for user to paste.

    **Wire into main.go:**
    - Add `root.AddCommand(jumpCmd)` alongside existing commands.
