// IsBranchMerged checks if branchName has been merged into targetBranch.
// Returns true if branchName is an ancestor of targetBranch (exit code 0).
// Returns false if not an ancestor (exit code 1 -- not an error).
// If the branch ref does not exist at all, returns true (branch was cleaned up post-merge).
func IsBranchMerged(ctx context.Context, repoPath, branchName, targetBranch string) (bool, error)
```

Implementation:
1. First check if branchName ref exists: `run(ctx, repoPath, "rev-parse", "--verify", "refs/heads/"+branchName)`. If error, also try `refs/remotes/origin/"+branchName`. If neither exists, return true (branch deleted = assumed merged).
2. Call `run(ctx, repoPath, "merge-base", "--is-ancestor", branchName, targetBranch)`.
3. Exit code 0 -> return true, nil
4. Exit code 1 -> return false, nil (use errors.As to check *exec.ExitError, ExitCode()==1)
5. Any other error -> return false, err

type ResolvedTask struct {
    Task      Task
    IsReady   bool     // all deps completed AND branches merged
    IsBlocked bool     // at least one dep incomplete or branch unmerged
    BlockedBy []string // IDs of blocking upstream tasks
    Layer     int      // topological layer (0 = no deps)
}

func Resolve(tasks []Task, deps []dependency.Dep, branchMerged map[string]bool) ([]ResolvedTask, error)
```

**RED phase -- write tests first** (resolver_test.go):

Table-driven tests using the pattern `tests := []struct{ name string; tasks []Task; deps []dependency.Dep; branchMerged map[string]bool; wantErr bool; check func(t *testing.T, result []ResolvedTask) }`:

1. "no tasks" -- empty input, empty output
2. "single task no deps" -- one queued task -> ready=true, layer=0
3. "completed task" -- one completed task -> ready=false, blocked=false, layer=0
4. "linear chain A->B" -- A completed, B queued -> B is ready, layer 1
5. "linear chain A->B, A incomplete" -- A queued, B queued -> B is blocked, BlockedBy=[A.ID]
6. "diamond A->C, B->C" -- A completed, B completed, C queued -> C ready, layer 1
7. "diamond partial" -- A completed, B queued, C queued -> C blocked by B
8. "branch not merged blocks" -- A completed (has branch), branchMerged[A.ID]=false, B depends on A -> B blocked
9. "branch merged allows" -- A completed, branchMerged[A.ID]=true, B depends on A -> B ready
10. "running task can be blocked" -- A queued, B running depends on A -> B blocked
11. "cycle detection" -- A->B->A -> returns error
12. "three layers" -- A->B->C -> layers 0, 1, 2

**GREEN phase -- implement Resolve()**:

Kahn's algorithm for layered topological sort:
1. Build adjacency map: upstream[taskID] = list of dependency task IDs
2. Compute in-degree for each task
3. Initialize queue with in-degree 0 tasks (layer 0)
4. Process queue layer by layer: dequeue all current layer, decrement in-degree of downstream tasks, add newly zero-degree tasks as next layer
5. If processed count != len(tasks), there's a cycle -> return error
6. For each resolved task, compute ready/blocked:
   - Completed tasks: ready=false, blocked=false
   - Otherwise: check all upstream deps. If any upstream is NOT completed OR (has branch AND branchMerged[upstream.ID]==false), then blocked=true, add to BlockedBy
   - If not blocked and state is "queued": ready=true
