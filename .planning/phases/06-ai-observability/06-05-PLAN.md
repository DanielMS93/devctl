  <action>
Create `internal/agent/patch.go`:

1. `CheckPatch(ctx context.Context, repoPath, patchData string) error`:
   - Write patchData to a temp file
   - Run `git apply --check <tempfile>` in repoPath
   - Return nil if clean, error with stderr if conflicts
   - Clean up temp file

2. `ApplyPatch(ctx context.Context, repoPath, patchData string) error`:
   - First call CheckPatch to verify
   - Write patchData to temp file
   - Run `git apply <tempfile>` in repoPath
   - Return nil on success, error with stderr on failure
   - Clean up temp file

3. `RevertPatch(ctx context.Context, repoPath, patchData string) error`:
   - Write patchData to temp file
   - Run `git apply --reverse <tempfile>` in repoPath
   - Return nil on success, error with stderr on failure
   - Clean up temp file

4. Helper `writeTempPatch(patchData string) (string, error)`:
   - Write to os.CreateTemp with `.patch` suffix
   - Return path for caller to defer os.Remove

Create `internal/agent/patch_test.go`:
   - Test CheckPatch with a valid patch against a temp git repo
   - Test ApplyPatch applies changes to working tree
   - Test RevertPatch undoes applied changes
   - Test CheckPatch returns error for invalid/conflicting patch
   - Setup: create a temp directory, `git init`, create a file, `git add`, `git commit`, generate a valid diff

2. `agentReviewCmd` — `devctl agent review [patch-id]`:
   - If patch-id provided: show that specific patch (full diff output)
     - Get patch from PatchStore (supports UUID prefix match)
     - Print title, status, branch, created_at
     - Print patch_data (the diff content)
   - If no patch-id: list all draft patches in a table format
     - Columns: ID (8-char prefix), Title, Branch, Status, Created
   - Use DB from cobra context (same dbKey pattern as task.go)

3. `agentApplyCmd` — `devctl agent apply <patch-id>`:
   - Get patch from PatchStore
   - Verify status is "draft" or "approved" (error if already applied/reverted)
   - Call `agent.CheckPatch` first — if conflicts, print error and exit
   - Call `agent.ApplyPatch`
   - Update patch status to "applied" via PatchStore.SetApplied
   - Print success message with patch title

4. `agentRevertCmd` — `devctl agent revert <patch-id>`:
   - Get patch from PatchStore
   - Verify status is "applied" (error if not applied)
   - Call `agent.RevertPatch`
   - Update patch status to "reverted" via PatchStore.SetReverted
   - Print success message

5. `agentConfigCmd` — `devctl agent config`:
   - Load and print current AgentConfig as formatted output
   - Show enabled/disabled status for each workflow
   - Show threshold and cooldown values

6. `agentToggleCmd` — `devctl agent toggle <workflow>`:
   - Toggle the enabled state of a workflow in viper config
   - Call `viper.Set("agent.workflows.<workflow>.enabled", !current)`
   - Write config with `viper.WriteConfig()`
   - Print new state

Register in main.go: add `rootCmd.AddCommand(agentCmd)` with subcommands.
