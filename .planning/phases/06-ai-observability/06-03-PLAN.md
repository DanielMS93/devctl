Create `internal/claude/watcher.go`:

1. Define `JSONLEntry` struct with fields parsed from each JSONL line:
   - `Type string` — "user", "assistant", "system", "progress"
   - `Timestamp time.Time`
   - `ToolName string` — for assistant entries with tool_use
   - `ToolTarget string` — command or file path
   - `UserText string` — for user entries, the text content
   - `Raw string` — original line for fallback display

2. Define `JSONLTailer` struct:
   - `path string` — JSONL file path
   - `offset int64` — last read position
   - `ctx context.Context`
   - `cancel context.CancelFunc`
   - `Entries chan JSONLEntry` — buffered channel (capacity 64)

3. `NewJSONLTailer(ctx context.Context, path string) *JSONLTailer`:
   - Create child context with cancel
   - Initialize offset to current file size (only show NEW entries, not history)
   - Return tailer with buffered channel

4. `(t *JSONLTailer) Run()` goroutine:
   - 500ms ticker
   - On each tick: stat file, if size > offset, open file, seek to offset, scan new lines
   - Parse each new line with `parseEntry(line) JSONLEntry`
   - Send to Entries channel (non-blocking: skip if full)
   - Update offset to new file size
   - Use open-read-close pattern (no persistent file handles, per research pitfall 2)
   - Exit on ctx.Done()

5. `(t *JSONLTailer) Stop()` — calls cancel

6. `parseEntry(line string) JSONLEntry`:
   - Unmarshal to `map[string]any`
   - Extract type, timestamp
   - For assistant type: extract tool_use name + target (same logic as scanner's extractRecentTools)
   - For user type: extract text content
   - Set Raw to original line

Create tests in `watcher_test.go`:
   - Test parseEntry with sample JSONL lines
   - Test NewJSONLTailer initializes offset to file size
   - Test Run() picks up appended lines (write to temp file, verify entry arrives on channel)

Create `pkg/tui/panels/session_viewer.go`:

1. Define `SessionViewer` struct (follows ViewerModel pattern — plain struct, not tea.Model):
   - `Visible bool`
   - `width, height int`
   - `vp viewport.Model` (from bubbles/v2/viewport)
   - `lines []string` — accumulated formatted log lines
   - `autoScroll bool` — true by default, false when user scrolls up
   - `sessionID string` — for display
   - `sessionSlug string` — for display
   - `tailerEntries <-chan claude.JSONLEntry` — receive channel from tailer
   - `tailerCancel func()` — to stop the tailer on close

2. `NewSessionViewer() SessionViewer` — returns hidden viewer.

3. `(v *SessionViewer) Open(sessionID, slug, jsonlPath string) tea.Cmd`:
   - Set Visible=true, autoScroll=true, clear lines
   - Create a new JSONLTailer for the given path
   - Start the tailer goroutine (via a tea.Cmd that launches it)
   - Store tailerEntries channel and cancel function
   - Return a tea.Cmd that starts polling the tailer channel (see step 5)

4. `(v *SessionViewer) Close()`:
   - Call tailerCancel to stop the tailer
   - Set Visible=false, clear state

5. Polling pattern — return a `tea.Cmd` that reads from the tailer channel:
   ```go
   func (v *SessionViewer) pollTailer() tea.Cmd {
       ch := v.tailerEntries
       return func() tea.Msg {
           entry, ok := <-ch
           if !ok { return sessionViewerClosedMsg{} }
           return sessionViewerEntryMsg{entry: entry}
       }
   }
   ```

6. `(v *SessionViewer) Update(msg tea.Msg) (bool, tea.Cmd)`:
   - Handle `sessionViewerEntryMsg`: format entry as `[HH:MM:SS] ToolName target` line, append to lines, rebuild viewport content, re-arm pollTailer. If autoScroll, scroll to bottom.
   - Handle `sessionViewerClosedMsg`: close viewer.
   - Handle `tea.KeyPressMsg`:
     - Esc/q: close viewer, return consumed=true
     - up/k, down/j, pgup, pgdown: forward to viewport. If user scrolls up from bottom, set autoScroll=false. If user scrolls to bottom, set autoScroll=true.
     - Other keys: forward to viewport
   - Return (consumed bool, cmd tea.Cmd) same pattern as ViewerModel.Update

7. `(v SessionViewer) View() string`:
   - Header: session slug or ID, styled blue
   - Viewport content
   - Help line: "Esc=close  ↑↓=scroll  (auto-scroll: on/off)"
   - Wrapped in border like ViewerModel

8. Format entries as readable log lines:
   - User: `[HH:MM:SS] USER: text...` (green)
   - Assistant tool_use: `[HH:MM:SS] [ToolName] target` (yellow for Bash, cyan for Read/Write/Edit)
   - System/other: `[HH:MM:SS] (type)` (dim)
   - Truncate long commands/paths to fit width

Wire into root.go:
1. Add `sessionViewer panels.SessionViewer` field to RootModel
2. In NewRootModel: `sessionViewer: panels.NewSessionViewer()`
3. In Update, add sessionViewer message handling BEFORE the viewer handling (same pattern):
   ```go
   if m.sessionViewer.Visible {
       consumed, cmd := m.sessionViewer.Update(msg)
       if consumed { return m, cmd }
   }
   ```
4. Add 'l' key binding (for "live"): when in right panel with a session selected, open session viewer:
   - Get selected session from rightPanel
   - Determine JSONL path: `claude.ClaudeProjectDir(session.ProjectPath) + "/" + session.ID + ".jsonl"`
   - Call `m.sessionViewer.Open(session.ID, session.Slug, jsonlPath)`
   - Set log bar status "Live session viewer (Esc to close)"
5. In View(): check `m.sessionViewer.Visible` and render it in place of right panel (same overlay pattern as viewer/taskGraph)
6. In propagateSizes: pass dimensions to sessionViewer

Update logs.go: add 'l=live view' to the hint bar text.

Add `sessionViewerEntryMsg` and `sessionViewerClosedMsg` as unexported types in session_viewer.go.
