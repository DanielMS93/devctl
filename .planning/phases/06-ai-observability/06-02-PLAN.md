Add to `internal/claude/scanner.go`:

1. Define `ToolActivity` struct: `Tool string` (e.g. "Bash", "Read", "Write", "Edit", "Agent"), `Target string` (command or file path), `Timestamp time.Time`.

2. Add fields to `Session` struct:
   - `CurrentTool string` — name of the most recent tool_use with no subsequent user entry (indicates still executing)
   - `CurrentCommand string` — target of the current tool (file path for Read/Write/Edit, truncated command for Bash)
   - `RecentTools []ToolActivity` — last 5 tool activities

3. Add function `extractRecentTools(lines []string, maxTools int) []ToolActivity`:
   - Scan lines from the end backwards
   - For each line with `type == "assistant"`, extract `message.content` array
   - For each content element with `type == "tool_use"`, extract:
     - `name` field as Tool
     - From `input`: for "Bash" use `command` (truncate to 80 chars); for "Read"/"Write"/"Edit" use `file_path` falling back to `path`; for "Agent" use `subagent_type`
     - Parse `timestamp` field as time.Time
   - Return up to maxTools activities (most recent first)
   - Use defensive `map[string]any` parsing (same pattern as existing parseJSONL)

4. Update `parseJSONL` to also call `extractRecentTools(lines, 5)` and determine `currentTool`/`currentCommand`:
   - After extracting recentFiles, call extractRecentTools
   - To determine "currently executing": find the index of the last `user` type entry. If the last `assistant` entry with tool_use is AFTER the last `user` entry, that tool is "currently executing". Otherwise currentTool is empty.
   - Return the additional fields (change return signature to return a `sessionExtras` struct or add them to the function)

5. Update `ScanSessionsWithThreshold` to populate the new Session fields from parseJSONL output.

Add tests in `scanner_test.go`:
- Test extractRecentTools with sample JSONL lines containing tool_use entries
- Test that currentTool is set only when last assistant entry is after last user entry
- Test defensive parsing: missing fields, unknown tool names handled gracefully
