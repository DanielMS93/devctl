Add to `internal/git/git.go`:
- `LastCommitTime(ctx context.Context, repoPath, branch string) (time.Time, error)`: runs `git log -1 --format=%ct <branch>` in repoPath. Parse Unix timestamp. Return zero time if branch has no commits or ref not found.

Create `internal/agent/idle.go`:

1. Define `IdleBranch` struct: `RepoPath string`, `Branch string`, `IdleSince time.Time`.

2. Define `IdleTracker` struct:
   - `lastActivity map[string]time.Time` â€” branchKey -> last known activity
   - `triggered map[string]time.Time` â€” branchKey -> when we last triggered
   - `config AgentConfig`

3. `NewIdleTracker(cfg AgentConfig) *IdleTracker`.

4. `(t *IdleTracker) Check(states []tuimsg.WorktreeState, commitTimes map[string]time.Time) []IdleBranch`:
   - For each worktree state:
     - Compute branchKey as `repoPath + ":" + branch`
     - Skip if repo is in config.DisabledRepos
     - Compute latest activity as max of: session LastActivity times, commitTime for branch (from commitTimes map), polledAt as fallback
     - Update lastActivity map
     - If `time.Since(latest) > config.IdleThreshold()` AND (not triggered OR triggered more than config.Cooldown() ago):
       - Mark triggered[branchKey] = time.Now()
       - Append to idle list
     - If `time.Since(latest) < config.IdleThreshold()`:
       - Reset triggered[branchKey] (delete from map, activity resumed)
   - Return idle branches

5. `(t *IdleTracker) ResetBranch(repoPath, branch string)`: manually reset trigger state (for when activity resumes or user intervenes).

Create `internal/agent/idle_test.go`:
   - Test that a branch with no activity for > threshold returns as idle
   - Test that a branch within threshold does NOT return as idle
   - Test cooldown: second call within cooldown period does not re-trigger
   - Test cooldown reset: activity resuming then going idle again triggers
   - Test disabled repos are skipped

Create `internal/agent/runner.go`:

1. Define `WorkflowRunner` struct:
   - `runStore *AgentRunStore`
   - `patchStore *PatchStore`
   - `config AgentConfig`

2. `NewWorkflowRunner(runStore *AgentRunStore, patchStore *PatchStore, cfg AgentConfig) *WorkflowRunner`.

3. `(r *WorkflowRunner) RunWorkflows(ctx context.Context, idle IdleBranch)`:
   - For each enabled workflow in config.Workflows:
     - Check cooldown: call runStore.LastTriggered to see if this workflow ran for this branch recently
     - Create AgentRun record (status=running)
     - Run the workflow command via `exec.CommandContext` with 5-minute timeout:
       - Shell: `sh -c <command>` with working directory set to idle.RepoPath
       - Capture stdout+stderr
     - On success: if stdout is non-empty and looks like a diff (contains "---" and "+++"), create an AgentPatch record with patch_data=stdout, status=draft, title=workflow name
     - On failure: update AgentRun status=failed with error_msg
     - On success: update AgentRun status=completed
   - Check config.MaxPatchSizeKB â€” reject patches larger than limit

4. `(r *WorkflowRunner) RunAsync(ctx context.Context, idle IdleBranch)`:
   - Launches RunWorkflows in a goroutine (NOT from TUI Update â€” from Manager only)
   - This is safe because Manager owns background goroutines

Integrate into `internal/dashboard/manager.go`:
1. Add fields to Manager:
   - `idleTracker *agent.IdleTracker`

   - `patchStore *agent.PatchStore`
   - `runner *agent.WorkflowRunner`
