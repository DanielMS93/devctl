Add a `statusIndicator` function that derives a visual status from existing `tuimsg.WorktreeState` fields:

```go
func statusIndicator(wt tuimsg.WorktreeState) string {
    // Running: has at least one active Claude session
    for _, s := range wt.Sessions {
        if s.IsActive {
            return "●" // running (will be colored green)
        }
    }
    // Blocked: has merge conflicts (UU status in changed files)
    for _, cf := range wt.ChangedFiles {
        if cf.StagedStatus == 'U' || cf.UnstagedStatus == 'U' {
            return "!" // blocked (will be colored red)
        }
    }
    return "○" // idle
}
```

Update `renderWorktreeRow` to prepend the status indicator before the branch name. Apply color: green ("2") for "●", red ("1") for "!", dim ("240") for "○". The indicator replaces the leading spaces before the cursor, so the row format becomes:

```

Keep the existing `buildStats` function unchanged -- the `◆N` session count stays in stats as supplemental info. The status indicator is the primary "at a glance" status.

Do NOT add new fields to tuimsg.WorktreeState -- derive status purely from existing fields (Sessions, ChangedFiles). "Finished" and "interrupted" statuses are Phase 5 dependencies; do not implement them.

func (v *ViewerModel) OpenDiff(worktreePath, filePath string) tea.Cmd {
    v.Visible = true
    v.worktreePath = worktreePath
    v.filePath = filePath
    v.diffMode = 0
    v.showingDiff = true
    v.vp = viewport.New(viewport.WithWidth(v.width-4), viewport.WithHeight(v.height-6))
    return v.loadDiffContent()
}
```

This mirrors the existing `Open` method but sets `showingDiff = true` and calls `loadDiffContent()` instead of `loadFileContent()`. The existing `d` key inside the viewer already cycles through diff modes, so this just provides a direct entry point.
